how I work

Architecture first. Code second.

Security decisions made during design cost nothing. Made during remediation, they cost everything. Every project follows the same discipline.

01

Define the threat model

What are we protecting? Who is the adversary? What's the blast radius of a breach? Documented before a line of code is written.

02

Design the architecture

Network topology, IAM boundaries, encryption decisions, and data flows. Diagrammed in Excalidraw or Mermaid.

03

Write infrastructure as code

Terraform modules for every resource. No ClickOps. State in S3 with DynamoDB locking.

04

Harden, test, and monitor

CIS benchmarks applied. Detection rules validated. Dashboards live before the project is called done.

vpc_security.tf
# private subnets only - no public egress by default
resource "aws_vpc" "main" {
  cidr_block = "10.0.0.0/16"
  enable_dns_hostnames = true
  enable_dns_support = true
}
 
# deny all inbound/outbound on default security group
resource "aws_default_security_group" "default" {
  vpc_id = aws_vpc.main.id
  # no ingress or egress rules
}
 
# flow logs to CloudWatch for network visibility
resource "aws_flow_log" "vpc" {
  vpc_id = aws_vpc.main.id
  traffic_type = "ALL"
  iam_role_arn = aws_iam_role.flow_log.arn
  log_destination = aws_cloudwatch_log_group.vpc_flow.arn
}

infrastructure

This module deploys a VPC with three private subnets, enables VPC Flow Logs to CloudWatch, and locks down the default security group. No public subnets. No NAT gateway unless you need one.

terraform-provisioned aws security baseline
detection

GuardDuty fires a finding. EventBridge picks it up. SNS routes it to a Lambda that normalizes the payload and posts to PagerDuty. The whole pipeline is Terraform - no ClickOps, no drift.

guardduty + cloudtrail: detection pipeline from scratch
architecture

AWS-only was a deliberate call. One control plane, one permission model, one billing boundary. Multi-cloud sounds good in a blog post. In practice, it is complexity that compounds at every layer.
